RecombinantGrowth

Lean 4 Extension — Combinatorial Explosion of Ideas under Free Recombination

Source: IdeaTheory/Extensions/RecombinantGrowth.lean

Overview

The RecombinantGrowth extension formalises the combinatorial explosion of ideas under free binary recombination. It operates entirely within the Finset API of Mathlib, so every result is computable and decidable. Two operators drive the theory:

  • binaryClosure(s) — the smallest set containing \(s\) and closed under one round of binary composition: \[ \operatorname{binaryClosure}(s) \;=\; s \;\cup\; igl\{\,a \circ b \;\mid\; a,b \in s\,igr\}. \] One application of binaryClosure takes a pool of \(n\) ideas to a pool of at most \(n + n^2\) ideas (Theorem T9).
  • closureN(s, n) — the \(n\)-step iterated closure, defined by \(\operatorname{closureN}(s, 0) = s\) and \(\operatorname{closureN}(s, n+1) = \operatorname{binaryClosure}(\operatorname{closureN}(s,n))\). This is the algebraic model of the Jones (1995) “ideas beget ideas” equation.

The 21 theorems proved here collectively establish Weitzman’s combinatorial wealth hypothesis as a theorem: super-exponential growth is a structural consequence of the composition operation, not an empirical regularity. Non-rivalry is algebraic — the closure always contains the originals — and the bounds on cardinality growth (\(\le n + n^2\) from above; \(\ge 3\) from two genuinely distinct elements below) delimit the space of Weitzman’s “combinatorial wealth.”

All results assume a type \(lpha\) equipped with an IdeaTheoryStructure instance, which provides the binary composition \(\circ\), an identity element \(e\), and the axioms id_left/id_right. DecidableEq is required for Finset operations. No further algebraic axioms are needed: the theorems in this file hold for any magma-like structure satisfying the identity laws.

Core Definitions

Pool, binaryClosure, closureN

-- Pool and binary closure definitions
abbrev Pool (α : Type*) [DecidableEq α] [IdeaTheoryStructure α] : Type _ := Finset α

def binaryClosure (s : Finset α) : Finset α :=
  s ∪ (s ×ˢ s).image (fun p => op p.1 p.2)

def closureN (s : Finset α) : Nat → Finset α
  | 0     => s
  | n + 1 => binaryClosure (closureN s n)

Pool α is a type alias for Finset α when \(lpha\) carries an IdeaTheoryStructure. The binaryClosure definition unpacks directly into the Lean 4 Finset API: s ×ˢ s is the Cartesian product, .image applies the composition \(\circ\), and ∪ adjoins the originals. The image step may collapse duplicates (if \(a \circ b = a \circ c\) for distinct \(b,c\)), which is why the upper bound in T9 uses the cardinality of the Cartesian product rather than the number of distinct pairs.

The closureN recursor is definitionally transparent: the pattern match is structural, so closureN_succ reduces to rfl. This matters for proof automation — many downstream lemmas are proved by simp only [closureN_succ] followed by binaryClosure_mono.

Membership Characterisation

-- Membership characterization
lemma mem_binaryClosure {s : Finset α} {x : α} :
    x ∈ binaryClosure s ↔ x ∈ s ∨ ∃ a ∈ s, ∃ b ∈ s, op a b = x

This lemma is the workhorse of the file. Almost every subsequent theorem is proved by rewriting via mem_binaryClosure and then performing case analysis. The left disjunct covers “original ideas”; the right disjunct covers “recombinant ideas.”

Non-Rivalry and Explosion Signatures

-- Non-rivalry: originals always survive
theorem subset_binaryClosure (s : Finset α) : s ⊆ binaryClosure s
theorem subset_closureN (s : Finset α) (n : Nat) : s ⊆ closureN s n
-- Combinatorial explosion
theorem binaryClosure_card_le (s : Finset α) :
    (binaryClosure s).card ≤ s.card + s.card * s.card
theorem recombinant_explosion {s : Finset α} {a b : α}
    (ha : a ∈ s) (hb : b ∈ s) (hab : op a b ∈ s)
    (hne1 : a ≠ b) (hne2 : op a b ≠ a) (hne3 : op a b ≠ b) : 3 ≤ s.card

Theorems and Lemmas

All 21 results are stated below in the order they appear in the Lean file. Each theorem box includes the Lean name, the mathematical statement, and a proof sketch.

Membership and One-Step Structural Properties

Theorem (mem_binaryClosure). \(x \in \operatorname{binaryClosure}(s) \;\iff\; x \in s \; ee\; \exists\,a \in s,\;\exists\,b \in s,\; a \circ b = x.\)
Proof sketch: Unfold binaryClosure using union and image; simp with mem_image and mem_product.
Theorem (subset_binaryClosure). \(s \subseteq \operatorname{binaryClosure}(s).\) Every idea in the pool is in its own recombinant closure (non-rivalry).
Proof sketch: Finset.mem_union.mpr with Or.inl.
Theorem (binaryClosure_mono_step). \(\operatorname{binaryClosure}(s) \subseteq \operatorname{binaryClosure}(\operatorname{binaryClosure}(s)).\)
Proof sketch: Apply subset_binaryClosure to binaryClosure(s).
Theorem (binaryClosure_mono). \(s \subseteq t \;\Rightarrow\; \operatorname{binaryClosure}(s) \subseteq \operatorname{binaryClosure}(t).\) The closure operator is monotone.
Proof sketch: Case analysis using mem_binaryClosure; existing elements use h, compositions use Or.inr with h applied to both components.

Iterated Closure: closureN

Theorem (closureN_succ). \(\operatorname{closureN}(s,\, n+1) = \operatorname{binaryClosure}(\operatorname{closureN}(s,n)).\) The recurrence unfolding for the \(n\)-step closure.
Proof sketch: By definition (rfl).
Theorem (closureN_subset_succ). \(\operatorname{closureN}(s,n) \subseteq \operatorname{closureN}(s,n+1).\) The recombinant trajectory is monotonically increasing.
Proof sketch: Apply subset_binaryClosure to closureN(s,n).
Theorem (subset_closureN). \(s \subseteq \operatorname{closureN}(s,n)\) for all \(n \in \mathbb{N}\). The original pool is always contained in every later closure (non-rivalry in iterated form).
Proof sketch: Induction on \(n\); zero is refl; succ chains the induction hypothesis with closureN_subset_succ.
Theorem (closureN_mono). \(m \le n \;\Rightarrow\; \operatorname{closureN}(s,m) \subseteq \operatorname{closureN}(s,n).\) The closure is monotone in the time parameter.
Proof sketch: Induction on the \(\le\) proof; refl for the base case; the step case chains the induction hypothesis with closureN_subset_succ.

Cardinality Bounds

Theorem (binaryClosure_card_le). \(\operatorname{card}(\operatorname{binaryClosure}(s)) \le \operatorname{card}(s) + \operatorname{card}(s)^2.\) One round of recombination grows the pool by at most \(n^2\) new ideas.
Proof sketch: card_union_le; card_image_le for the composition image; card_product gives \(n^2\).
Theorem (card_le_insert). \(\operatorname{card}(s) \le \operatorname{card}(\operatorname{insert}\,a\,s).\) Inserting an idea never decreases the pool.
Proof sketch: Finset.card_le_card with Finset.subset_insert.

Special Cases: Singleton and Empty

Theorem (binaryClosure_singleton_ident). \(\operatorname{binaryClosure}(\{e\}) = \{e\}.\) The identity singleton is closed under recombination.
Proof sketch: ext; simp with mem_binaryClosure; forward direction: the only element is \(e\), and composition of \(e\) with \(e\) is \(e\) by id_left; backward direction: Or.inl.
Theorem (binaryClosure_empty). \(\operatorname{binaryClosure}(\emptyset) = \emptyset.\) The empty pool stays empty.
Proof sketch: Unfold binaryClosure; simp.

Explosion and Erasure

Theorem (recombinant_explosion). If \(a, b \in s\), \(a \circ b \in s\), \(a e b\), \(a \circ b e a\), \(a \circ b e b\), then \(\operatorname{card}(s) \ge 3.\) A pool containing two distinct ideas with a genuinely new composition has at least 3 elements (Romer / Weitzman explosion).
Proof sketch: Show \(\{a,\,b,\,a \circ b\} \subseteq s\) and compute its cardinality as 3 using the distinctness conditions.
Theorem (erase_card_lower_bound). \(\operatorname{card}(s) \le \operatorname{card}(s.\operatorname{erase}\,a) + 1.\) Removing one idea removes at most one element from the pool (non-rivalry in cardinality form).
Proof sketch: Case on \(a \in s\): use card_erase_of_mem; otherwise erase is identity.
Theorem (binaryClosure_erase_subset). \(\operatorname{binaryClosure}(s.\operatorname{erase}\,a) \subseteq \operatorname{binaryClosure}(s).\)
Proof sketch: Apply binaryClosure_mono to Finset.erase_subset.
Theorem (closureN_erase_subset). \(\operatorname{closureN}(s.\operatorname{erase}\,a,\,n) \subseteq \operatorname{closureN}(s,\,n).\) Erasing one idea before iterating still yields a sub-pool of the full iteration.
Proof sketch: Induction on \(n\); zero uses erase_subset; succ uses the induction hypothesis and binaryClosure_mono.

Identity, Closure Characterisation, and Union

Theorem (binaryClosure_union_ident). \(\operatorname{binaryClosure}(s \cup \{e\}) = \operatorname{binaryClosure}(s) \cup \{e\}.\) Adjoining the identity commutes with one round of recombination.
Proof sketch: ext; case analysis on membership using mem_binaryClosure; compositions involving \(e\) reduce via id_left/id_right.
Theorem (op_mem_binaryClosure). \(a \in s \;\wedge\; b \in s \;\Rightarrow\; a \circ b \in \operatorname{binaryClosure}(s).\) Any composition of pool elements is in the closure.
Proof sketch: mem_binaryClosure.mpr with Or.inr.
Theorem (op_mem_of_binaryClosure_eq). \(\operatorname{binaryClosure}(s) = s \;\Rightarrow\; a \in s \;\Rightarrow\; b \in s \;\Rightarrow\; a \circ b \in s.\) If \(s\) is its own closure (already closed under \(\circ\)), then compositions stay in \(s\).
Proof sketch: Apply op_mem_binaryClosure; rewrite with the fixed-point hypothesis.
Theorem (binaryClosure_eq_iff_closed). \(\operatorname{binaryClosure}(s) = s \;\iff\; orall\,a \in s,\; orall\,b \in s,\; a \circ b \in s.\) The closure is a fixed point if and only if the pool is already closed under composition.
Proof sketch: Forward direction: op_mem_of_binaryClosure_eq. Backward direction: Finset.Subset.antisymm using the closure hypothesis and subset_binaryClosure.
Theorem (union_binaryClosure_subset). \(\operatorname{binaryClosure}(s) \cup \operatorname{binaryClosure}(t) \subseteq \operatorname{binaryClosure}(s \cup t).\) The closure of the union dominates the union of closures.
Proof sketch: Finset.union_subset; each component uses binaryClosure_mono with subset_union_left/subset_union_right.

Implications for Growth Theory

Non-rivalry is not an assumption in Idea Theory — it is a theorem. The subset_binaryClosure theorem proves that every original idea survives every round of recombination. This formalises the economic intuition behind Romer’s growth model: unlike physical capital, ideas are not consumed by use. The algebraic mechanism is transparent: binaryClosure(s) is defined as s ∪ (compositions), so the union contains \(s\) as a subset by construction. But this means non-rivalry is not merely a definitional artifact — it is a structural property of the closure operator that persists under iteration (as subset_closureN proves) and under erasure of individual ideas (as closureN_erase_subset proves). Non-rivalry is robust.

The recombinant_explosion theorem proves Weitzman’s combinatorial wealth claim from first principles. The proof requires only three distinct elements — \(a\), \(b\), and \(a \circ b\) — with three distinctness conditions. No continuous mathematics, no probability, no economic equilibrium assumptions. The result is purely combinatorial: the composition operation on a set of ideas necessarily generates new elements, provided the compositions are genuinely novel. This is the algebraic mechanism behind the super-linear growth of scientific knowledge: each new idea that is genuinely compositionally distinct from existing ones increases the pool’s recombinant potential by more than linearly.

The binaryClosure_eq_iff_closed theorem identifies precisely when creative recombination is exhausted: when the idea pool is already closed under composition. A closed pool is its own closure — adding recombination to it changes nothing. This is the algebraic characterisation of a “mature” or “paradigmatic” scientific field in Kuhn’s sense: normal science operates within a closed idea pool, generating recombinations that are already implicitly contained in the paradigm. Revolutionary science — the creation of genuinely novel ideas — corresponds to the addition of elements that break the closure: new ideas \(a\) such that \(a otin \operatorname{binaryClosure}( ext{old pool})\). The theorem makes this precise.

The monotonicity and subset theorems collectively prove that the recombinant growth of an idea pool is path-dependent and irreversible. closureN_mono shows growth is non-decreasing in time. subset_closureN shows original ideas are permanently accessible. binaryClosure_mono shows that larger initial pools produce larger closures at every step. This path-dependence is not a feature of any particular growth model — it is a structural consequence of the monotonicity of the closure operator. Ideas that have been generated cannot be “un-generated.” The history of a pool’s recombinant trajectory is fully determined by its initial contents and the composition operation — there is no stochasticity, no optimisation, no equilibrium selection. Just the algebra.

Connections to the Idea Zoo

Romer — Non-Rival Ideas and Endogenous Growth

Paul Romer’s endogenous growth theory (1990) built on two premises: ideas are non-rival (one person’s use of an idea does not prevent another’s) and ideas combine to generate new ideas. The subset_binaryClosure theorem (T2) is the algebraic proof of non-rivalry: after any round of recombination, every original idea remains in the pool. It has not been “used up.” The subset_closureN theorem (T7) extends this to all time steps: non-rivalry is not just a one-step property but a permanent feature of the closure operator. The erase_card_lower_bound theorem (T14) gives the cardinality version: excising a single idea removes at most one element — the marginal cost of idea use is at most 1, regardless of how many others use the same idea. Romer’s growth model thus rests on a theorem, not merely an assumption.

Weitzman — Combinatorial Wealth

Martin Weitzman (1998) argued that the diversity of ideas produces “combinatorial wealth” — the number of new ideas generated by recombination grows super-linearly with the initial stock. The binaryClosure_card_le theorem (T9) gives the upper bound: one round of recombination grows the pool from \(n\) to at most \(n + n^2\) elements. The recombinant_explosion theorem (T13) provides the lower bound in a concrete case: a pool of 2 elements with a genuinely new composition already has 3 elements. The gap between the upper and lower bounds — \(n^2\) minus 1 — is the space in which Weitzman’s “combinatorial wealth” lives. The closureN_mono theorem (T8) shows that the wealth is non-decreasing over time. Weitzman’s intuition — that a larger idea stock generates combinatorially more recombinant possibilities — is validated by these two bounds taken together.

Jones — Idea-Based Growth

Charles Jones (1995, 2019) formalised Romer’s model and emphasised that the growth rate of the idea stock depends on the existing stock: ideas beget ideas. The closureN_succ recurrence (T5) is the algebraic version of this self-referential growth: \(\operatorname{closureN}(s, n+1) = \operatorname{binaryClosure}(\operatorname{closureN}(s,n))\). Each new round of recombination operates on the already-expanded pool, not the original. The closureN_mono theorem (T8) shows that this process is monotone: closures at later times dominate earlier ones. Jones’s “semi-endogenous growth” models — where the growth rate depends on the population of researchers — correspond to the interaction between rogers_monotone (in Diffusion) and closureN_mono here. The union_binaryClosure_subset theorem (T21) captures Jones’s “standing on the shoulders of giants” effect: merging two idea pools and recombining dominates recombining them separately.

Hayek — The Knowledge Problem

Friedrich Hayek (The Use of Knowledge in Society, 1945) argued that knowledge is dispersed across individuals and cannot be aggregated without loss. The binaryClosure_union_ident theorem (T17) formalises a Hayekian insight: adjoining the identity element — the “null knowledge” of an agent who knows nothing — commutes with recombination. Agents who contribute no knowledge (hold the identity idea) neither add to nor subtract from the recombinant closure. The closureN_erase_subset theorem (T16) shows that removing one agent’s knowledge always yields a sub-pool of the full closure: no single agent is indispensable to the recombinant process. Hayek’s dispersal thesis is partially refuted at the level of closure: the union of individual idea pools has a richer recombinant closure than any subset. But no central aggregator is needed — the closure operator is purely compositional.

Schumpeter — Creative Destruction

Joseph Schumpeter (Capitalism, Socialism and Democracy, 1942) identified innovation as the engine of capitalist growth, driven by the recombination of existing elements into new configurations — “creative destruction.” The op_mem_binaryClosure theorem (T18) formalises the creation side: any composition of pool elements is automatically in the closure. The op_mem_of_binaryClosure_eq theorem (T19) formalises the destruction side in its most abstract form: if a pool is already closed, then combining elements within it stays within it — the pool is “saturated” and there is no further creative destruction. Schumpeter’s dynamic occurs precisely when the pool is not closed — when \(\operatorname{binaryClosure}(s) e s\). The binaryClosure_eq_iff_closed theorem (T20) gives the exact criterion: creative potential is exhausted if and only if the pool is already closed under composition.

Arrow — Knowledge as a Public Good

Kenneth Arrow (Economic Welfare and the Allocation of Resources for Invention, 1962) argued that knowledge has the properties of a public good: non-excludability and non-rivalry. The non-rivalry proof (subset_binaryClosure) has already been discussed. The union_binaryClosure_subset theorem (T21) captures non-excludability in its mathematical form: the closure of the union of two idea pools contains (at least) the union of their individual closures. Sharing an idea pool strictly dominates withholding: \[ \operatorname{binaryClosure}(s \cup t) \;\supseteq\; \operatorname{binaryClosure}(s) \cup \operatorname{binaryClosure}(t), \] with the gap representing the cross-recombinant ideas that can only be generated when both pools are accessible. Arrow’s argument for public funding of basic research corresponds exactly to this theorem: the recombinant surplus from open access to idea pools cannot be achieved by any private arrangement that keeps them separate.

Proof Technique Notes

The Role of mem_binaryClosure

The single most important lemma in the file is mem_binaryClosure (T1). Nearly every subsequent proof proceeds by rewriting the membership condition and then performing a case split on the two disjuncts. The left disjunct (\(x \in s\)) is handled by the monotonicity of the surrounding operator; the right disjunct (\(\exists a,b \in s,\; a \circ b = x\)) is handled by supplying witnesses. This two-branch pattern is so uniform that several proofs reduce to one-liners after unfolding.

Induction on \(n\) vs Induction on \(\le\)

Two styles of induction appear in the file. The closureN lemmas (T7, T8) use different induction targets: subset_closureN (T7) inducts on the natural number \(n\), while closureN_mono (T8) inducts on the proof of \(m \le n\). The latter style is more expressive: it allows the base case to handle \(m = n\) (reflexivity) and the step case to chain with closureN_subset_succ. Both proofs are short; the choice is a matter of which induction principle gives the cleaner structure.

Cardinality Arguments

The cardinality bound in T9 is the sharpest easily provable upper bound. The key steps are: (1) Finset.card_union_le gives \(\operatorname{card}(A \cup B) \le \operatorname{card}(A) + \operatorname{card}(B)\); (2) Finset.card_image_le gives \(\operatorname{card}(\operatorname{image}\,f\,A) \le \operatorname{card}(A)\); and (3) Finset.card_product gives \(\operatorname{card}(s imes s) = \operatorname{card}(s)^2\). Chaining these three produces the bound in one calculation. The bound is not tight: if the composition is idempotent (\(a \circ a = a\) for all \(a\)) then many compositions collapse, and the actual growth may be much smaller. The binaryClosure_singleton_ident (T11) and binaryClosure_empty (T12) theorems exhibit the extreme cases.

The Fixed-Point Criterion

Theorem T20 (binaryClosure_eq_iff_closed) is the most structurally significant result in the file because it characterises when recombination adds nothing. The backward direction (“if closed then equal”) requires Finset.Subset.antisymm: the \(\supseteq\) direction is subset_binaryClosure (T2); the \(\subseteq\) direction requires showing every recombinant element is already in \(s\), which follows from the closure assumption. The forward direction is a direct application of T19. The criterion is the Lean analogue of the algebraic notion of a sub-magma: a set that is closed under the binary operation.
